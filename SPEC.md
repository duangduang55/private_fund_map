# Spec: 私募基金拓客系统 UI 优化方案

## 当前 UI 状态评估

经过全面扫描，当前 UI 已有较好的基础（企业蓝灰专业风格、暗色主题、统一色板），但存在以下**可优化项**：

### 已做好的（保留）
- ✅ 统一暗色主题（`#0f172a` bg / `#1e293b` card / `#2563eb` accent）
- ✅ 6px 圆角系统、一致的语义色（succcess/warning/danger）
- ✅ 状态标签（完成/待办/取消）配色体系完善
- ✅ 地图页 AUM 分色体系合理
- ✅ 专业克制，无花哨特效

### 可优化的痛点
| # | 问题 | 严重程度 | 影响 |
|---|------|---------|------|
| 1 | 无共享 CSS 文件，9 个文件各自维护一套样式 | ⚠️ 高 | 维护成本高，一致性难保证 |
| 2 | 字体栈不一致（HTML 模板 vs Streamlit） | ⚡ 中 | 细微的视觉断裂感 |
| 3 | Emoji 代替图标（✅ ⭐ 🛒 🔍 📋 等） | ⚡ 中 | 不专业，不同系统渲染差异大 |
| 4 | 输入框背景色不一致（`rgba(255,255,255,0.04)` vs `#1e293b`） | 💡 低 | 视觉不一致 |
| 5 | card padding 不统一（0.8rem~2rem 跨度） | 💡 低 | 间距系统无约束 |
| 6 | 响应式设计有限（仅 2 个页面有断点） | ⚠️ 高 | 移动端/平板体验差 |
| 7 | 缺少加载/空状态/错误状态视觉反馈 | ⚡ 中 | UX 完整度不足 |
| 8 | Streamlit 使用大量 `!important` 覆盖默认样式 | 💡 低 | 样式脆弱 |
| 9 | 打印样式仅 confirm.html 有 | ⚡ 中 | 其他页面导出体验差 |
| 10 | 缺少平滑过渡和微交互动效 | 💡 低 | 交互感受略显生硬 |

---

## 优化目标

**保持现有风格**（企业蓝灰专业风、暗色主题），集中解决上述 10 个痛点，实现：

1. **一致性** — 建立共享设计系统，消除所有不一致
2. **专业感** — 图标替换 emoji，统一字体栈，完善交互细节
3. **可用性** — 响应式、加载态、空状态、错误态全覆盖
4. **可维护性** — CSS 变量体系、共享样式文件、减少重复

## 设计系统方案

### 配色方案（保留现有，扩展语义）

```
--bg-primary:         #0f172a   页面背景
--bg-secondary:       #1e293b   卡片/面板背景
--bg-tertiary:        #1e3a5f   信息提示背景
--border-default:     #334155   默认边框
--border-input:       #475569   输入框边框
--border-focus:       #3b82f6   聚焦边框
--text-primary:       #f1f5f9   主文本
--text-secondary:     #94a3b8   次要文本
--text-muted:         #64748b   灰色信息
--text-tertiary:      #475569   页脚/禁用
--accent:             #2563eb   主色
--accent-hover:       #1d4ed8   主色悬停
--accent-light:       rgba(37,99,235,0.12)  主色浅底
--success:            #16a34a   成功
--success-light:      rgba(22,163,74,0.12)
--warning:            #d97706   警告
--warning-light:      rgba(217,119,6,0.12)
--danger:             #dc2626   危险
--danger-light:       rgba(220,38,38,0.12)
```

保留不变，现有配色体系成熟，只做微调。

### 字体系统（统一）
- **HTML 模板/Streamlit 统一使用**：`-apple-system, BlinkMacSystemFont, "Segoe UI", Roboto, "Helvetica Neue", Arial, sans-serif`
- **字号层级**：h1 1.5rem(600) → h2 1.25rem(600) → h3 1rem(500) → body 0.875rem(400) → small 0.75rem(400)
- **行高**：body 1.5, headings 1.3, small 1.4

### 间距系统（8pt 网格）
- `--space-xs: 4px`  (0.25rem)
- `--space-sm: 8px`  (0.5rem)
- `--space-md: 12px` (0.75rem)
- `--space-lg: 16px` (1rem)
- `--space-xl: 24px` (1.5rem)
- `--space-2xl: 32px`(2rem)

### 圆角系统（保持统一）
- `--radius-sm: 4px`  — 标签/徽章
- `--radius-md: 6px`  — 卡片/按钮/输入框 ✅ 保持现状
- `--radius-lg: 8px`  — 模态框/大面板

### 阴影
- `--shadow-sm: 0 1px 2px rgba(0,0,0,0.3)` — 卡片
- `--shadow-md: 0 4px 6px rgba(0,0,0,0.4)` — 下拉/弹出
- `--shadow-lg: 0 10px 15px rgba(0,0,0,0.5)` — 模态框

### 过渡
- `--transition-fast: 150ms ease`
- `--transition-normal: 200ms ease`

## 功能点拆解

以下是按优先级排列的优化功能点：

### Phase 1: 基础架构（建立设计系统）

- [ ] **[UI] Task 1: 创建共享 CSS 文件** `static/style.css`
  - 提取所有 CSS 变量到 `:root`
  - 定义通用组件类（卡片、按钮、表单、标签、表格、徽章）
  - HTML 模板全部引入此文件
  - **验证：** 所有页面加载后 CSS 变量生效，基础组件样式统一

- [ ] **[UI] Task 2: Streamlit 注入 CSS 变量**
  - `app.py` 的 CUSTOM_CSS 改为引用 `static/style.css` 中的变量
  - 只保留 Streamlit 特有覆写
  - 消除 `!important` 滥用
  - **验证：** Streamlit 页面样式不退化

- [ ] **[UI] Task 3: 统一字体栈和间距**
  - 所有 HTML 模板统一字体 stack
  - .streamlit/config.toml 同步使用系统字体栈
  - 检查所有页面间距对齐 8pt 网格
  - **验证：** 所有页面字体一致，间距规范

### Phase 2: 图标系统

- [ ] **[UI] Task 4: 引入 SVG 图标系统**
  - 创建 `static/icons/` 或使用内联 SVG sprite
  - 定义常用图标：星标、购物车、地图、计划、反馈、PDF、搜索、删除、编辑等
  - 替换所有 emoji 图标引用
  - **验证：** 所有页面无 emoji 图标残留

### Phase 3: 页面打磨

- [ ] **[UI] Task 5: 优化地图页面** `map.html`
  - 搜索框/图例/统计/购物车面板样式打磨
  - 购物车滑出动画
  - Toast 通知样式优化
  - 标记弹出框内图标替换
  - 响应式适配（平板）
  - **验证：** 地图功能完整，视觉一致

- [ ] **[UI] Task 6: 优化确认页面** `confirm.html`
  - 命中原因卡片样式统一
  - 机构列表卡片布局优化
  - PDF 导出打印样式打磨
  - **验证：** 确认功能正常，打印效果良好

- [ ] **[UI] Task 7: 优化计划列表页** `plans.html`
  - 表格响应式（移动端卡片化）
  - 状态标签使用新图标系统
  - 筛选/搜索体验优化
  - **验证：** 列表功能完整，响应式表现正常

- [ ] **[UI] Task 8: 优化反馈表单页** `feedback.html`
  - 标签切换组件改用图标+文字
  - 表单布局对齐规范
  - 提交反馈动效
  - **验证：** 反馈提交功能完整

- [ ] **[UI] Task 9: 优化机构详情页** `detail.html`
  - 信息卡片层级优化
  - 标签系统替换
  - 拜访时间线样式统一
  - **验证：** 详情展示完整

### Phase 4: 全链路体验

- [ ] **[UI] Task 10: 统一加载态/空状态/错误态**
  - 定义通用 Loading spinner（CSS 实现，无图片依赖）
  - 空状态插画/提示文案
  - 错误提示样式统一
  - 所有页面添加状态反馈
  - **验证：** 模拟加载/空/错误场景，样式正确

- [ ] **[UI] Task 11: 响应式适配**
  - 地图页：平板横屏适配
  - 确认页：小屏卡片堆叠
  - 计划列表：Mobile 端 table → cards
  - 反馈页：表单单列布局
  - 详情页：信息卡片自适应
  - **验证：** 375px / 768px / 1440px 视口下布局不破损

- [ ] **[UI] Task 12: 微交互动效**
  - 卡片 hover 轻微上浮（`translateY(-1px)` + shadow 变化）
  - 按钮点击反馈（scale 或 color shift）
  - 页面切换/数据加载过渡
  - **验证：** 动效自然不突兀，`prefers-reduced-motion` 兼容

### Phase 5: 打印优化

- [ ] **[UI] Task 13: 全页面打印样式**
  - 为所有 HTML 模板添加 `@media print` 样式块
  - 隐藏交互元素（按钮、导航、搜索框等）
  - 确保表格/卡片内容完整打印
  - **验证：** 各页面打印预览正常

## 执行计划

### 工作流

```
Phase 1 (基础架构) → Phase 2 (图标系统) → Phase 3 (页面打磨) → Phase 4 (全链路体验) → Phase 5 (打印优化)
```

每个阶段内任务按编号顺序执行。

### 验证方法

每个任务完成后：
1. 运行 `python3 -m py_compile` 检查语法
2. 运行 `ruff check` 检查代码风格
3. 手动检查页面渲染效果
4. 更新 HARNESS_PROGRESS.md

### 风险与缓解

| 风险 | 影响 | 缓解 |
|------|------|------|
| CSS 共享化可能遗漏全局选择器冲突 | 中 | 逐页测试，保留回退 |
| SVG 图标替换 emoji 需全面搜索 | 低 | grep 搜索所有模板 |
| 响应式改动可能影响现有布局 | 中 | 每次修改后手动检查关键页面 |

### 已确认的设计决策

| 决策项 | 选择 | 
|--------|------|
| 图标方案 | 内联 SVG Sprite |
| 状态标识 | SVG 图标 + 中文文字标签 |
| 响应式断点 | 375px(手机) / 768px(平板) / 1440px(桌面) |
