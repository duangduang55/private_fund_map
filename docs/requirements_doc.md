# 私募基金拓客辅助系统 — 需求文档

## 1. 项目背景

项目团队通过走访私募机构开展拓客业务，面临以下问题：

- **数据不同步**：团队成员各自拜访私募机构，拜访记录分散，存在重复拜访情况
- **工具脱节**：现有的地图看板（基于 Excel 快照）和查询应用（基于数据库）各自独立，数据源不一致
- **缺少工作流**：从筛选目标 → 查看分布 → 制定拜访计划 → 记录反馈，没有完整的闭环流程

因此需要将现有工具整合升级为团队协作的**私募拓客辅助系统**，统一数据源、统一拜访计划、记录拜访全流程。

## 2. 项目目标

构建一套支持多用户协作的私募拓客辅助系统，实现：

1. 从私募数据库中按多维度条件筛选目标机构
2. 筛选结果可在地图上直观查看分布，辅助规划拜访路线
3. 支持"待拜访清单（购物车）→ 确认计划 → 反馈记录"的完整工作流
4. 已拜访机构在地图上有特殊标记，点击可查看历史拜访信息，避免重复拜访
5. 支持团队协作，区分普通成员和管理员权限

## 3. 用户角色

| 角色 | 权限 | 说明 |
|------|------|------|
| **admin** | 全部权限 | 可查看所有人的拜访计划和反馈记录，管理用户 |
| **member** | 有限权限 | 可查询数据、使用地图、创建自己的拜访计划和反馈，仅看自己的记录 |

## 4. 功能需求

### 模块一：用户认证（Phase 1）

| 编号 | 功能 | 说明 |
|------|------|------|
| F1.1 | 用户登录 | 用户名 + 密码登录，JWT 保持会话 |
| F1.2 | 用户注册 | admin 可创建新用户（暂不开放自助注册） |
| F1.3 | 密码管理 | bcrypt 哈希存储密码 |
| F1.4 | 会话保持 | JWT token 设定有效期，过期重新登录 |

### 模块二：私募数据查询与导出（Phase 1，基于现有功能升级）

| 编号 | 功能 | 说明 |
|------|------|------|
| F2.1 | 多维筛选 | 办公省份、管理规模、企业性质、会员类型、机构类型、产品数量范围、机构名称模糊查询、产品名称模糊查询（多关键词 OR）、高管履历模糊查询（多关键词 OR） |
| F2.2 | 排除已清算产品 | 默认开启的开关 |
| F2.3 | 结果表格展示 | 中文列名展示所有字段，智能显示命中列 |
| F2.4 | Excel 导出 | 含时间戳文件名，导出列含实际命中的命中列 |
| F2.5 | 数据来源 | 从 `ts_quant_db.private_fund_list` + `private_fund_detail`（只读）查询 |

### 模块三：动态地图展示（Phase 1-2）

| 编号 | 功能 | 说明 |
|------|------|------|
| F3.1 | 查询结果一键显示到地图 | 在查询结果页点击"在地图上显示"，打开动态地图页面展示筛选出的机构 |
| F3.2 | 图例切换 | 地图左下角图例支持按**管理规模**和**是否拜访过**两种维度切换显示，切换后地图标记动态更新 |
| F3.3 | 拜访状态标记区分 | 已拜访的私募使用灰色 + 特殊边框标记，区别于未拜访的按规模颜色标记 |
| F3.4 | 搜索定位 | 地图上提供搜索框，输入关键词模糊匹配机构名称，匹配到的机构标记有视觉反馈（高亮/跳动/显示名称） |
| F3.5 | 标记点击弹窗 | 点击任意标记弹出详情窗口，包含：机构名称、基金数量、管理规模、搜索命中类型（高管履历命中/产品名称命中）、详情页URL链接、"加入待拜访清单"按钮 |
| F3.6 | 已拜访标记弹窗扩展 | 如果该私募已被拜访过，弹窗额外显示：此私募最近由[拜访人]于[时间]拜访过，[摘要] |
| F3.7 | 地图瓦片源 | 使用高德地图瓦片（已验证国内加载流畅） |

### 模块四：待拜访清单与计划确认（Phase 2）

| 编号 | 功能 | 说明 |
|------|------|------|
| F4.1 | 加入待拜访清单 | 从地图标记弹窗点击"加入待拜访清单"，直接加入当前用户的购物车（visit_cart 表） |
| F4.2 | 侧滑清单窗口 | 地图右侧侧滑出清单列表，每条显示：机构名称、管理规模、基金数量、办公地址，每条记录有删除按钮 |
| F4.3 | 进入确认 | 侧滑窗口底部有"进入确认"按钮，点击后跳转到确认页面 |
| F4.4 | 确认页面 | 展示待确认清单列表，可继续删除记录。需填写：拜访日期、拜访人姓名 |
| F4.5 | 提交计划 | 确认提交后，购物车中的条目逐条生成 visit_plans 记录（状态为 pending），购物车清空 |
| F4.6 | 返回修改 | 确认页面支持"返回上一步"回到清单编辑 |

### 模块五：拜访反馈（Phase 3）

| 编号 | 功能 | 说明 |
|------|------|------|
| F5.1 | 反馈入口 | 从拜访计划列表中点击"填写反馈"进入 |
| F5.2 | 反馈表单 | 拜访日期（预填计划日期）、拜访人（预填）、拜访状态（成功/未见到/已搬迁/地址有误/其他）、是否获取联系方式（是/否）、摘要、沟通情况描述、后续跟进建议、标签（合作意向/不对外/不合作/需求高度匹配，可多选） |
| F5.3 | 反馈提交 | 提交后更新 visit_plans 状态为 completed，写入 visit_feedback 表 |
| F5.4 | 反馈编辑 | 已提交的反馈可编辑修改 |

### 模块六：拜访历史与管理（Phase 3）

| 编号 | 功能 | 说明 |
|------|------|------|
| F6.1 | 拜访计划列表 | 展示所有 visit_plans 记录，可按日期范围/拜访人/状态筛选 |
| F6.2 | 权限控制 | admin 可见全部记录，member 仅见自己的记录 |
| F6.3 | 状态切换 | admin 可将 cancelled 状态的计划恢复或删除 |
| F6.4 | 统计概览 | admin 首页展示：本月拜访数、完成率、各状态分布等统计 |

## 5. 非功能需求

| 编号 | 需求 | 说明 |
|------|------|------|
| N1 | 响应速度 | 查询响应 < 3 秒（正常数据量下），地图加载 < 5 秒 |
| N2 | 地图流畅度 | 标记点 < 5000 个时无明显卡顿 |
| N3 | 安全性 | 全部 SQL 参数化查询防注入；密码 bcrypt 哈希；JWT 鉴权 |
| N4 | 可部署性 | 支持 Docker 容器化部署，配置通过环境变量注入 |
| N5 | 可维护性 | 代码分层清晰；数据库连接信息集中配置；关键操作有日志 |

## 6. 技术架构

### 6.1 整体架构

```
┌────────────────────────────────────────────────────────────────┐
│                        Nginx (反向代理)                          │
└──────────┬─────────────────────────────────────┬────────────────┘
           │                                     │
           ▼                                     ▼
┌──────────────────────┐          ┌──────────────────────────────┐
│   Streamlit (8501)   │          │     FastAPI (8000)           │
│                      │          │                              │
│  · 用户登录           │   HTTP   │  · REST API (JSON)           │
│  · 数据查询/筛选      │◄────────►│  · JWT 认证                  │
│  · 表格展示/Excel导出  │          │  · 动态地图页面 (HTML+JS)    │
│  · 拜访计划/反馈表单   │          │  · 购物车 CRUD               │
│  · 拜访历史查看        │          │                              │
└──────────┬────────────┘          └──────────┬───────────────────┘
           │                                  │
           ▼                                  ▼
┌──────────────────────┐          ┌──────────────────────────────┐
│   ts_quant_db (只读)  │          │   fund_map_db (读写)         │
│  · private_fund_list  │          │  · users                    │
│  · private_fund_detail│          │  · visit_cart               │
│                       │          │  · visit_plans              │
│                       │          │  · visit_feedback           │
└──────────────────────┘          └──────────────────────────────┘
```

### 6.2 关键技术选型

| 组件 | 技术 | 说明 |
|------|------|------|
| 查询/表单前端 | Streamlit | Python 原生，快速构建数据应用 |
| 地图前端 | Leaflet + 高德瓦片 | 国产地图源，加载流畅 |
| API 后端 | FastAPI | 轻量异步，自动生成 OpenAPI 文档 |
| 数据库 | PostgreSQL 18 | 已有 Docker 运行中 |
| 认证 | JWT (python-jose) + bcrypt | 无状态 token |
| 部署 | Docker Compose | 本地可跑，方便上云 |

### 6.3 数据源规范

| 数据库 | 用户 | 权限 | 用途 |
|--------|------|------|------|
| `ts_quant_db` | `db_query_user` | 只读 SELECT | 私募机构数据（爬虫来源） |
| `fund_map_db` | `fund_map_user` | 全部 CRUD | 用户、拜访计划、反馈等业务数据 |
